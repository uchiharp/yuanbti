## 二、文件一：代号鸢人物关系网

### ✅ 做得好的

1. **viewport 设置正确**：`width=device-width,initial-scale=1,maximum-scale=1,user-scalable=no`，避免双击缩放干扰
2. **100dvh 适配**：使用 `height:100dvh` 兼容移动浏览器地址栏收起
3. **touch-action:manipulation**：消除 300ms 点击延迟
4. **-webkit-tap-highlight-color:transparent**：去除 iOS 点击高亮
5. **图例交互**：点击可筛选关系类型，有 off 状态视觉反馈（透明度降低 + 删除线）
6. **侧边面板**：点击节点弹出关联列表，使用 CSS transform 滑入动画
7. **力导向图交互**：支持 roam（缩放平移）+ draggable（拖拽节点）
8. **emphasis focus adjacency**：高亮关联节点和边，暗淡无关节点，这是关系网的最佳实践
9. **底部提示**：初始显示"点击角色查看关联人"，首次引导明确

### ⚠️ 需要改进

#### P0 - 严重影响体验

1. **overlay 缺少 pointer-events**
   - `#overlay.show{display:block}` 但没有设置 `pointer-events:auto`，默认继承 `display:none` 的状态
   - **影响**：打开面板后，点击遮罩层可能无法关闭面板
   - **修复**：`#overlay.show{display:block;pointer-events:auto;background:rgba(0,0,0,.4)}`

2. **图例在手机端遮挡内容**
   - `position:fixed;top:8px;left:8px;max-width:180px`，在 375px 屏幕上占比近 50%
   - 深色半透明背景 + 12px 字体，关系类型列表可能很长
   - **建议**：默认折叠，只显示标题"关系类型"，点击展开；或将图例移到底部抽屉

3. **节点标签在 375px 下严重重叠**
   - `fontSize:11` 对力导向图节点标签偏大，40 个节点在手机屏幕上必然密集重叠
   - **建议**：手机端默认隐藏标签，仅在缩放到一定程度或 hover/focus 时显示

#### P1 - 明显影响体验

4. **面板关闭按钮太小**
   - `font-size:22px;padding:4px 8px`，触摸热区约 30x30px，低于 44px 推荐标准
   - **建议**：增大 padding 到 `8px 12px`，或使用更大的关闭图标

5. **面板内容缺少滚动回弹**
   - `overflow-y:auto` 在 iOS 上需要 `-webkit-overflow-scrolling:touch`
   - **建议**：添加 `overflow-y:auto;-webkit-overflow-scrolling:touch`

6. **无初始加载状态**
   - ECharts 内嵌后文件体积很大（~1MB+），首次渲染可能有明显白屏
   - **建议**：添加简单的 loading 指示器

7. **底部提示消失后无恢复途径**
   - 点击图表后 hint 通过 opacity 渐隐，用户忘了怎么操作时无法找回
   - **建议**：提供常驻的"?"帮助按钮

#### P2 - 锦上添花

8. **force 参数针对手机未优化**
   - `repulsion:280, edgeLength:[80,220]` 在 375px 屏上节点间距偏大，导致需要大量平移才能看全图
   - **建议**：根据屏幕宽度动态调整：`repulsion: 280 * (375/window.innerWidth)`

9. **关系面板中 count 列含义不明**
   - 数字 `49`、`45` 等代表什么？互动次数？剧情出场次数？缺少单位说明
   - **建议**：在面板 header 下方添加一行说明"数字表示剧情互动次数"

10. **图例筛选是 prefix 匹配，不够精确**
    - `l.split('·')[0]` 做前缀匹配，"辟雍同窗·挚友"和"辟雍同窗·暗流涌动"都被归为一类
    - 这可能不是 bug 而是设计意图，但应让用户知道

## 三、文件二：代号鸢世界线时间流

### ✅ 做得好的

1. **viewport 设置正确**（同上）
2. **散点图 + timeAxis 组合**：用时间轴控制剧情事件浏览，适合"时间流"概念
3. **emphasis focus adjacency**：同样配置了关联高亮
4. **scatter 系列 + visualMap**：用颜色/大小编码信息维度

### ⚠️ 需要改进

#### P0 - 严重影响体验

1. **时间轴在手机端几乎不可用**
   - ECharts timeline 组件设计为桌面端，播放/暂停按钮 + 刻度标签在 375px 下极其拥挤
   - 触摸滑动时间轴的操作与图表的 roam 行为可能冲突
   - **建议**：
     - 移除 ECharts 原生 timeline，改用底部水平滚动条 + 自定义控制器
     - 或添加一个底部时间选择器（类似 range slider）

2. **散点图在手机上触控精度不足**
   - 散点密集时，手指触摸面积（~44px）远大于散点（~10-20px symbolSize），难以精确选择
   - tooltip 的 `triggerOn:'mousemove'` 在移动端不会触发（没有 hover）
   - **建议**：`tooltip.triggerOn` 改为 `'click'`，并增大 symbolSize 的触摸热区

3. **缺少数据概览 / 入口引导**
   - 打开后只有一个散点图，用户不知道这些点代表什么、时间范围是什么
   - **建议**：顶部添加简短说明（如"横轴为时间，纵轴为世界线编号，点大小表示剧情量"）

#### P1 - 明显影响体验

4. **roam 在手机端的 pinch 缩放可能被浏览器拦截**
   - 没有看到明确的 `touch-action` 控制
   - **建议**：在 chart 容器上设置 `touch-action:pan-x pan-y pinch-zoom`

5. **visualMap 组件在手机端占用空间**
   - 连续型 visualMap 的拖拽手柄在触屏上操作困难
   - **建议**：改为分段选择器或下拉筛选

6. **缺少时间范围指示**
   - 用户不知道当前看到的是哪个时间段的数据
   - **建议**：在页面顶部固定显示当前时间窗口

## 四、跨文件通用问题

### 1. file:// 协议限制

- **XSS 安全策略**：某些浏览器对 file:// 下的 JS 有严格限制，确保没有使用 fetch/XMLHttpRequest
- **字体加载**：`-apple-system,PingFang SC` 是系统字体，OK；但如果用户设备没有这些字体会 fallback
- **建议**：添加更多 fallback 字体：`"PingFang SC","Hiragino Sans GB","Microsoft YaHei","WenQuanYi Micro Hei",sans-serif`

### 2. 无任何离线缓存策略

- 文件 >1MB，每次打开飞书下载都要重新传输
- **建议**：如果飞书支持，考虑使用 Service Worker 缓存（但 file:// 下 SW 不可用，可忽略）

### 3. 缺少页面标题和品牌识别

- `<title>` 有值但页面内无可见标题
- **建议**：在图表上方添加一行小标题（如"代号鸢 · 人物关系网"），加深色半透明背景

### 4. 无错误边界

- 如果 ECharts 初始化失败（极端情况），页面白屏无任何提示
- **建议**：包裹 try-catch，失败时显示"图表加载失败，请刷新重试"

## 六、一句话总结

关系网在桌面端体验良好，但手机端存在图例遮挡、标签重叠、面板关闭困难等问题；时间流在手机端几乎是"不可用"状态，需要重新设计时间轴交互方式。核心改进方向：**为 375px 做减法**——默认隐藏次要信息，用点击/长按代替 hover，用自定义组件替代 ECharts 桌面端控件。
