# 鸢BTI 出题指南（子Agent专用）v4.1

## 你的任务
为指定角色按批次出题，每道题必须基于真实剧情/台词。

## ⚠️ v4核心变化：单维度
每道题**只测1个维度**，3个选项对应 L(1)/M(3)/H(5)。
- dimension格式：`["S2"]`（单元素数组，保持向后兼容）
- scores格式：`{"S2": 1}`（只有1个key）
- 这样每个维度被独立测量，结果更准确

## 10维体系 + 3级评分
- S1权谋(权谋城府)、S2情感(情感表达)、S3金钱(金钱务实)、S4面具(面具深度)、S5行动(行动风格)、S6底线(底线弹性)、S7锋芒(表达锋芒)、S8温柔(温柔底色)、S9权力(权力态度)、S10秩序(秩序感)
- 评分只用：1、3、5
- 每题只覆盖1个维度

## 题目格式（JSON）

```json
{
  "id": "角色_批次_序号",
  "type": "sweet | funny | angst | scheme | daily | classic",
  "dimension": ["S2"],
  "source_character": "角色名",
  "text": "题干（3句话内）",
  "options": [
    {
      "label": "A",
      "text": "选项A（具体行动选择，体现S2低）",
      "scores": {"S2": 1},
      "tendency": "L",
      "reasoning": "不表达情感→S2:1"
    },
    {
      "label": "B",
      "text": "选项B（另一种反应，体现S2中等）",
      "scores": {"S2": 3},
      "tendency": "M",
      "reasoning": "间接表达→S2:3"
    },
    {
      "label": "C",
      "text": "选项C（第三种反应，体现S2高）",
      "scores": {"S2": 5},
      "tendency": "H",
      "reasoning": "直接表达情感→S2:5"
    }
  ],
  "reveal": "简短引用原文台词+场景解读"
}
```

## 🚨 reasoning字段（必须有）
每道题每个选项都必须有reasoning，格式：`"选项行为→维度得分原因"`
- 1句话，说清楚为什么这个选项在这个维度是这个分值
- 例：`"沉默不接话，回避情感交流→S2:1"`
- 例：`"笑着调侃但眼神认真，间接表达关心→S2:3"`
- 例：`"直接握住他的手说我陪你，毫无保留→S2:5"`

## 🚨 reasoning常见错误（v4.1新增，常犯）

### ❌ reasoning和分值矛盾
- **说"直接"但给最低分1** — "直接"通常对应高行动力/高表达，给1分矛盾
- **说"间接"但给最高分5** — "间接"通常对应中等分值，给5分矛盾
- 修复：要么改reasoning措辞，要么改分值

### ❌ reasoning太短无意义
- `"谋定后动"` — 没说清楚为什么是S5:1
- `"冲动先动"` — 没说清楚为什么是S5:5
- `"及时止损"` — 没说清楚对应什么维度什么分值
- 修复：至少写清楚"具体行为→维度:分值"

### ❌ reasoning和选项行为脱节
- reasoning说的是一回事，选项做的是另一回事
- 例：reasoning说"配合"但选项里没有配合的行为
- 例：reasoning说"回避"但选项其实是"沉默"或"转移话题"
- 修复：reasoning必须准确描述选项的行为，不要用抽象总结词

### ✅ 正确的reasoning写法
- 先写选项的具体行为（用了什么动作/说了什么话）
- 再写这个行为在目标维度上的含义
- 最后标分值
- 例：`"递给他一杯热水但没多说话，用行动照顾但保持距离→S8:3"`
- 例：`"当众模仿他的语气逗他，公开的锋芒不打压→S7:5"`

## 选项设计核心原则

### ❌ 禁止套公式
不要每道题都硬套「A=回避/保守 → B=中间 → C=大胆/激烈」。
不要每道题都是 A=1分 B=3分 C=5分（用户一眼看穿）。
选项方向由场景决定，同一批次5道题选项模式要有变化。
分值排列要随机打乱（6种排列：1-3-5、1-5-3、3-1-5、3-5-1、5-1-3、5-3-1）。

### ✅ 正确做法
每道题的3个选项从场景本身自然生长，是面对这个情境时3种合理但不同的反应。

**举例——甜蜜场景（测S2情感表达）**
- A: 低头不说话，假装没听到 → S2:1
- B: 笑着转移话题，但偷偷看了他一眼 → S2:3
- C: 直接说我也想你 → S2:5

**举例——权谋场景（测S1权谋城府）**
- A: 把情报直接给他，不做保留 → S1:1
- B: 交情报但暗中观察他的反应 → S1:3
- C: 先查清楚他的来路再决定交不交 → S1:5

**举例——搞笑场景（测S7锋芒）**
- A: 假装听不懂，尴尬一笑 → S7:1
- B: 反手一个冷笑话怼回去 → S7:3
- C: 当众拆穿，让所有人都听到 → S7:5

### 分值必须从选项含义推导
1. 这个选项描述了什么行为？
2. 这个行为在目标维度上是什么水平？（低1/中3/高5）
3. 写成reasoning

## 🚨 场景基调匹配（v4.1新增，常犯错误）

**搞笑场景不能当正经处理！** 如果原剧情是搞笑/玩梗氛围，选项必须保持搞笑基调。

### ❌ 错误示例
原剧情：周瑜说「广陵赚钱广陵花」，明显是搞笑场景
- A: 说不用了能应付 → S5:1（太正经）
- B: 说你打算怎么帮 → S5:3（太正经）
- C: 直接报了个数字 → S5:5（太正经）

### ✅ 正确做法
原剧情是搞笑的，选项也要有梗：
- A: 说花完了再赚，多大点事 → S5:1
- B: 说行那你先借我五百 → S5:3
- C: 说咱俩组个绣衣楼男团，靠脸吃饭 → S5:5

### 判断标准
- 原剧情台词本身是否搞笑/玩梗/夸张？
- 如果是，选项也必须有梗有趣，不能突然变严肃
- 搞笑场景测的维度可以不变，但选项风格必须匹配场景基调
- **日常场景(daily)也可以搞笑** — 日常不等于严肃，选项可以有轻松/调侃元素

## ⚠️ 台词说话对象（v4.1新增，常犯错误）

**台词的对象不一定是广陵王！** 出题前必须确认台词的说话对象。

### 已知踩坑
1. **张邈的侄女是呆呆** — 张邈说「呆呆是不是饿了」是在对呆呆说话，不是对广陵王。不能把广陵王代入呆呆的角色。
2. **郭嘉壶关相关台词** — 对象是贾诩/荀彧，不是广陵王。**广陵王没上过辟雍学宫，广陵王不在场。**不能包装成广陵王和郭嘉的互动场景。
3. **董奉制止拿弓** — 月海夜航船上，董奉制止的对象需要确认。

### 检查方法
出题时查台词上下文，确认这句台词的说话对象是谁：
```bash
db9 sql learn_test -q "SELECT speaker, text, line_order FROM dialogues WHERE story_title='故事名' AND category='剧情' ORDER BY line_order LIMIT 20"
```

## ⚠️ 广陵王在场性（v4.1新增，常犯错误）

**不是所有剧情广陵王都在场！** 出题时必须确认广陵王是否参与该场景。

### 已知不在场的场景
- 辟雍学宫相关剧情（广陵王没上过辟雍学宫）
- 郭嘉在学宫和贾诩/荀彧的对话
- 张邈和呆呆的互动
- **所有绒绒/动物剧情**（角色是动物形态，广陵王不在场）

### 检查方法
如果不确定广陵王是否在场，查上下文确认后再出题。

- 每个男主固定30题
- 维度分配：10维度各3题
- type分配：6种type各5题

### 密探
- 每个密探的题量按指定分配（见下方表格），不要多出
- 维度均衡看**所有密探加起来的总量**，不是每个密探单独均衡
- **密探type偏重funny/daily** — 不需要均匀分配6种type

## 6种type风格（参考）
1. **甜蜜暧昧（sweet）** — 暧昧互动、心动瞬间
2. **搞笑玩梗（funny）** — 有梗有趣、角色特有笑点
3. **扎心虐心（angst）** — 虐心、离别、牺牲、不安全感
4. **权谋博弈（scheme）** — 政治困境、权力抉择、算计
5. **日常温馨（daily）** — 日常互动、撒娇、生活细节
6. **名场面还原（classic）** — 经典剧情场景的行为选择

密探出题时从中选取合适的type，不需要全覆盖。

## ⚠️ 绝对禁止的事项

### 选项问题
1. **选项里不能塞对方角色的回应** — 选项是广陵王的选择
2. **选项不是行动，是观点/分析** — 必须是具体行为
3. **选项逻辑必须成立** — 不搞反角色关系
4. **选项要说人话** — 不文艺腔，不像AI写的
5. **选项不以"你"或"广陵王"开头** — 直接写动作
6. **选项不是完整主谓宾句子** — 可以省略主语
7. **选项分值和维度不能牵强** — 选项行为必须能自然推导出对应维度的分值

### 题干问题
8. **题干3句话内** — 只交代必要背景
9. **题干点明角色名字** — 用名字不写"他"
10. **不提章节名** — 包括「犬都」「朝歌之战」「燕歌行」「珠履三千」「七载相逢」「月海夜航」「犬都纪事」「犬都掌故」「犬都绒绒版」等活动名
11. **不出现剧情线标签** — 题干里不能出现「孙策线中」「傅融线」「左慈线」等剧情线标注，用户不需要知道这条台词来自哪条线
12. **不使用绒绒/动物剧情** — 「犬都纪事」「猫邦掌故」「犬都绒绒版」等活动中角色是动物形态（猫、狗等），不能用于真人BTI出题。题干里出现「喵」「汪」「围脖」「嗷呜」「大好狗」等动物化台词的，一定是绒绒剧情，必须跳过
11. **视角规则** — 90%广陵王视角，10%代入角色视角
12. **题干不能有其他角色的台词/反应** — 题干是背景描述，不是对话记录

### 人设问题
13. **人设解读忠于原文** — 不歪曲角色动机
14. **不OOC** — 广陵王不对男主说出冷漠粗鲁的话
15. **C选项可以大胆，可以性暗示，但不能粗鲁**

### 格式问题
16. **维度key只用S1-S10**
17. **分值只用1/3/5**
18. **type只允许**：sweet/funny/angst/scheme/daily/classic
19. **tendency只用**：L/M/H
20. **dimension必须是数组**：`["S2"]`
21. **⚠️ 分值判断的唯一依据是选项文本的具体行为内容** — 不能机械地用任何一个字段（scores/tendency/reasoning）去对齐另一个。正确做法：读题干→理解场景→读dimension→读每个选项的具体行为→判断该行为在目标维度上应该是什么水平（低1/中3/高5）。三个选项一起判断，确定相对排序后统一赋值{1,3,5}，再写scores+tendency+reasoning。
22. **⚠️ scores、tendency、reasoning三者必须一致** — 但一致性不是靠机械对齐实现的，而是三者的正确推导结果自然一致。出题时最后检查一遍：scores的分值 = tendency的映射(L=1,M=3,H=5) = reasoning末尾的分值标注。
23. **⚠️ 发现矛盾时的处理** — 不要批量暴力修改。必须逐题读场景+选项文本内容重新判断，不看旧值，独立推导出正确的分值，然后统一重写scores+tendency+reasoning。三个选项必须一起改，不能只改一个。

### 内容问题
21. **台词从数据库拉** — 用db9查询，不编造场景
22. **禁止AI味套话**
23. **有梗有趣** — 去掉角色壳子就不成立的才出

## 情商优化规则

### 什么时候需要调整
1. **必须改**：场景是安慰/关心/他人痛苦时，选项太冷漠/无情的
2. **可以改**：sweet/angst氛围中选项明显情商不合理的
3. **不改**：funny搞笑题（低情商反而好笑）、角色特色冷漠（如贾诩）、日常互动中合理的冷淡

### 原则
- 根据题目氛围调整，不是一刀切
- 保留角色特色——贾诩冷漠是正常的，不要强行改温柔
- 搞笑场景选项可以低情商，但要有梗
- 安慰场景中广陵王不能说太冷的话

## 🚨 台词引申含义（v4.2新增，常犯错误）

**很多台词不能只看表面意思！** 必须理解台词在剧情中的真正含义和上下文。

### 常见错误
- 阿蝉说「左右分还是上下分」→ 不是在讨论战术分工，是在讨论分头砍人（搞笑场景）
- 董奉说「被陈登钓鱼钓上来的」→ 不是真的钓鱼，是被陈登骗到徐州的
- 马超说「这辈子真带劲」→ 不是在开心，是在战场咳血说遗言（悲壮场景）

### 检查方法
出题前必须查台词上下文（前后10行），理解台词在剧情中的真实含义：
```bash
db9 sql learn_test -q "SELECT text, line_order FROM dialogues WHERE story_title='故事名' AND speaker='角色名' AND length(text) > 5 ORDER BY line_order"
```

### 规则
- 题干必须传达台词的正确含义，不能只引用表面文字
- 选项必须基于正确理解设计
- reasoning必须准确描述选项行为在场景中的含义
- 如果不确定台词含义，不要出这道题

## 🚨 密探type偏好（v4.2新增）

密探的type分配偏重funny/daily，不需要像男主那样均匀分配。
- 密探出题时优先选funny和daily
- angst/scheme/classic可以有但不是重点
- sweet可以有但密探的甜蜜和男主的甜蜜不同（更含蓄、更多是日常小细节）
