## 二、10维质心体系

### 评分制
7级评分：**1 / 2 / 2.5 / 3 / 3.5 / 4 / 5**

### 维度定义

| 维度 | 名称 | 低分倾向 | 高分倾向 |
|---|---|---|---|

| S2 | 情感 | 内敛含蓄，极少外露 | 热烈直接，毫无保留 |
| S3 | 金钱 | 不在意世俗价值 | 精打细算，极度务实 |
| S4 | 面具 | 表里如一，少有伪装 | 多重面孔，白切黑 |
| S5 | 行动 | 谋定后动，谨慎行事 | 冲动即决，凭直觉行动 |
| S6 | 底线 | 绝不越界，有绝对红线 | 不择手段，为目的可牺牲一切 |
| S7 | 锋芒 | 温和含蓄，不显露锋芒 | 毒舌犀利，刀刀见血 |

| 来源 | 每次出题数 |

## 四、动态分支系统

### 核心机制
- 题库500道在后台，每次测试只走其中一条路径
- 每道题的选项决定下一道题从哪个维度/题池抽取
- 类似决策树，允许多路径收敛

### 选择逻辑
1. 维度置信度 < 阈值 → 从该维度题池抽题
2. 维度置信度 ≥ 阈值 → 跳过，抽其他维度
3. 所有维度都够 → 进入收敛验证
4. 总题数达20-25道 → 输出结果

### 题目不重复原则
- 同一次测试中，每道题的情境/场景/问题角度不得重复
- 不同维度可以测相似主题，但切入角度必须有明显区别
- 同一角色台词在同一次测试中最多出现1次
- 同一故事背景不出现两次

### 重复可玩性
- 不同入口 → 不同路径 → 不同题目
- 同维度题池随机轮换
- 探索度追踪："你已探索 X/Y 条路径"

## 五、题目类型

| 类型 | 占比 | 说明 |
|---|---|---|
| 🎬 剧情情境题 | 50% | 从真实剧情提取冲突/抉择点 |
| 💭 价值观题 | 20% | 角色金句改编，选最共鸣的 |
| 🎭 密探日常题 | 15% | 非剧情的日常行为场景 |
| 🔮 二选一极端 | 15% | 角色原话当选项，强迫站队 |

### 🎬 剧情情境题（核心，占50%）

分两层：

**已知剧情题**（占剧情题60%）
- 来源：已公开主线/密传/恋念/活动剧情
- 可以直白，上下文给足，角色名保留
- 玩家有剧情情感记忆，代入感强
- 例：*「崇德殿上，李傕兵马逼宫。刘辩以天子之名震慑三军。如果是你——」*
- 答完可展示"如果你选A，和刘辩走了一样的路"

**匿名化剧情题**（占剧情题40%）
- 来源：同上，但去掉角色名和故事名
- 只给场景描述+冲突，测的是"你会怎么选"而不是"你知道谁选了什么"
- 避免玩家因为认识角色而"按角色选"
- 例：*「你手中握着一个能扳倒对手的秘密，但公开它会牵连无辜。你会——」*
- 答完揭晓"这个场景来自XX的XX剧情，你的选择和TA一样/不一样"

**出题方法：**
1. 从v5/v4分析文件提取角色的"关键剧情"
2. 按故事标题从数据库拉连续台词，还原剧情上下文
3. 找到冲突/抉择点 → 保留前因后果 → 改编成3选项
4. 选项对应维度的高/中/低分倾向

**约束：**
- 每道剧情题标注来源（角色+剧情线）
- 同一故事在同一测试中不重复出现
- 选项不能剧透关键反转，但要有足够上下文让玩家代入

### 💭 价值观题（20%）
- 用角色台词改编成态度选择题
- 例：*「面对不公，你更认同哪句？」* A"忍一时风平浪静" B"退让只会换来得寸进尺"
- 代号鸢台词本身有态度（张邈毒舌、陈登疯批、郭嘉反讽），直接用原话更有味道
- 不标注角色名，避免"按角色选"

### 🎭 密探日常题（15%）
- 非剧情抉择，日常感强的行为偏好题
- 例：*「修罗场里你怎么分配时间」「收到密信你先看谁的」「绣衣楼里你更想负责什么」*
- 融入代号鸢世界观（绣衣楼、密探体系、广陵王身份）
- 让玩家想象"如果我生活在那个世界里"

### 🔮 二选一极端（15%）
- 强迫站队，没有中间选项
- 直接用角色金句/经典台词当选项
- 例：*「以下两句话，你更认同哪句？」*
  A. "道德是铡刀，会切掉你的手"（郭嘉）
  B. "被咬断了，变成浮木，每天飘在水面上"（陈登）
- 快速区分性格底层逻辑
- 答完展示原台词出处和角色

## 六、城池进度系统

### 核心概念
测试 = 从广陵出发的一段旅程，每到一座城池回答几道题，然后启程下一站。玩家以「广陵王」身份经历。

### 历史与游戏约束（⚠️ 严格遵守）
- 所有城池名称、行政区划必须符合东汉末年（约184-220年）
- 城池描述不得出现东汉之后的制度、建筑、地名变更
- 称谓、官职、地理归属需符合东汉建制
- **必须与代号鸢游戏世界观一致**：
  - 城池中出现过的剧情、势力归属、密探相关事件
  - 融入游戏中的势力（绣衣楼、各路诸侯、广陵王势力等）
  - NPC、地名、机构名以游戏内设定为准
  - 城池氛围描述与游戏中呈现的风格保持一致

### 四条路线（均从广陵出发，每次随机选一条）

| 路线 | 城池顺序 | 方向 | 终点 |
|---|---|---|---|
| **西蜀线** | 广陵 → 寿春 → 许县 → 洛阳 → 成都（益州） | 西进 | 成都·山隐鸢阁仙门 |
| **幽州线** | 广陵 → 下邳 → 平原 → 邺城 → 幽州 | 北上 | 幽州 |
| **交趾线** | 广陵 → 长沙 → 桂阳 → 日南 → 交趾 | 南下 | 交趾 |
| **西凉线** | 广陵 → 寿春 → 许县 → 长安 → 西凉 | 西北 | 西凉 |

> 西蜀线和西凉线前两站重合（广陵→寿春→许县），后面分叉。

### 每座城池体验

| 站点 | 题数 | 体验 |
|---|---|---|
| 第1站（广陵） | 5道 | 轻松入门 |
| 第2站 | 7道 | 渐入佳境 |
| 第3站 | 8道 | 开始纠结 |
| 第4站 | 9道 | 深度探索 + 二选一 |
| 第5站（终点） | 8道 | 收敛验证 + 终极二选一 |
| 第6站（揭晓） | 3道 | 最后确认 + 出结果 |

**6座城 × 40道**

### 城池附加设计
- 每座城池有简短氛围描述（地理、军事地位、当时所属势力、游戏内剧情关联）
- 路线随机，每次走不同路遇到不同题
- 终点城池揭晓结果
- 答完后可回顾"旅程地图"
- 每座城池结束时给一句角色台词作为过渡

## 七、算法系统

### 维度选题逻辑

**状态追踪：**
- 每个维度记录：`{confidence: 0, values: []}`
- 答一题 → 记录具体分值（1/2/2.5/3/3.5/4/5）→ confidence+1
- 最终维度得分 = values 的加权平均（近端权重更高，终点站题权重×1.5）
- 维度倾向标签：均值<2.5→L，2.5-3.5→M，>3.5→H

**题型配额约束（单次30道）：**
- story_known: 9道（30%）
- story_anon: 6道（20%）
- value: 6道（20%）
- daily: 5道（17%）
- extreme: 4道（13%）
选题时先按维度置信度选维度，再按剩余配额选题型。

**选题规则：**
1. 找置信度最低的维度 → 从该维度题池抽题
2. 如果有2个以上维度并列最低 → 随机选一个
3. 题型轮换：不连续出同类型题（剧情→日常→金句→剧情...）
4. 角色轮换：同一角色不出连续题
5. 第4站开始：插入二选一极端题
6. 第5站：只出置信度仍<2的维度验证题 + 终极二选一

**终极二选一（仪式感）：**
- 找置信度最接近的两个维度 → 出一道同时测这两个维度的二选一
- 用这道题分出胜负，决定最终倾向
- 例：*「在权谋和温柔之间，你更偏哪边？」*

**收敛条件：**
- 所有维度置信度 ≥ 2 → 可以出结果
- 总题数达30道 → 强制出结果

### 准确性保障机制

**1. 选项随机排序**
- 不是固定L/M/H对应A/B/C，每次加载题目时随机打乱选项顺序
- 防止玩家发现"选C总是高分"的模式

**2. 反向题检测**
- 同一维度出两道方向相反的题，检测答题一致性
- 例：一道"你会主动出击吗"（高S5=5）+ 一道"你会先观察再行动吗"（低S5=1）
- 如果同一维度两道方向相反的题都选了同一档，标记该维度置信度+0.5（惩罚不专注）
- 在题目JSON中用 `reverse_pair` 字段标注配对关系

**3. 注意力陷阱题**
- 偶尔出一道"表面测A维度、实际测B维度"的题
- 例：题目描述很感人（以为测S2情感），但选项核心是"你选择行动还是忍耐"（实际测S5行动）
- 在题目JSON中用 `trap_dimension` 字段标注表面维度，`dimension` 标注实际测的维度
- 过滤掉不看选项就选的玩家

**4. 交叉维度题**
- 一道题同时给两个维度打分，提高信息密度
- 例：选项A同时给S1=4和S2=2，选项B同时给S1=2和S2=4
- 在题目JSON中用 `cross_dimension` 字段标注第二维度
- 30题中约5-8道为交叉维度题

**5. 首题校准**
- 第1道题用极端场景快速锚定玩家的大致范围
- 如果首题选了最极端的选项（H），后续该维度出题范围缩小到中高档（2.5/3.5/5）
- 如果首题选了中间选项（M），后续范围拉大（1/3/5）
- 校准后的题目分值范围在 `calibration_range` 字段中标注

**6. 热身题不计分**
- 前2道题是热身题，让玩家理解答题逻辑，不计入维度得分
- 标注 `warmup: true`，算法跳过计分
- 但用来校准难度：如果热身题都选极端，后续该维度可以少出确认题

**7. 选项描述均衡**
- 所有题目的选项文字长度差不超过1.5倍（不能一个5字一个50字）
- 选项情感倾向均衡：不能A明显正能量B明显负面
- 出题时自动检查，不达标则重写选项
- 在验证脚本中加入长度检查

**地理关联（可选）：**
- 每座城池优先出与该城相关的角色题
- 例：洛阳优先出袁基/刘辩题，邺城优先出郭嘉/曹操阵营题
- 不强制，题池不够时从全局抽

### 三种匹配算法

**1. 镜像匹配（你性格最像谁）**
- 欧氏距离：d = √(Σ(user[i] - char[i])²)
- 归一化相似度：similarity = 1 / (1 + d)
- 取相似度最高的Top3
- 核心：距离最近，各维度接近

**2. 互补匹配（谁会被你吸引）**
- 互补度 = Σ(|user[i] - char[i]|) × weight[i]
- S2情感、S8温柔权重1.2（高情感情柔角色更容易被吸引）
- 其余维度权重1.0
- 取互补度最高的Top3
- 核心：差异最大，互补最强

**3. 暗面匹配（你最不想承认的那一面）**
- 找和用户在某些极端维度上一致、但其他维度完全相反的角色
- 算法：取用户得分最高（>3.5）的维度，在这些维度上和用户接近但整体方向相反的角色
- 例：你S1权谋很高但S2情感很低 → 暗面可能是郭嘉（权谋高+情感低，但你们在其他维度背道而驰）
- 解读：*「你最不想承认的是——在XX这件事上，你和郭嘉一模一样」*

**三者区别：**
| 匹配类型 | 算法 | 含义 | 典型结果 |
|---|---|---|---|
| 镜像 | 欧氏距离 | 你最像谁 | 各维度接近的角色 |
| 互补 | 加权曼哈顿 | 谁会被你吸引 | 差异互补的角色 |
| 暗面 | 极端维度交集 | 你最不想承认的 | 某个极端面相似但整体相反的角色 |

## 八、结果呈现（组合模式）

### 三层结果

**第一层：你是什么类型的人**
- 基于10维评分，给出一个人格类型名（如「主权博弈者」「暴烈温柔型」「清醒疯子型」）
- 类型命名风格：代号鸢世界观感，不是冷冰冰的字母组合
- 附带一句话画像（如"你是用面具守护真心的权谋家"）
- 维度雷达图展示10维得分

**第二层：三种匹配**
- 🪞 镜像匹配：你性格最像谁（Top3）
- 💕 互补匹配：谁会被你吸引（Top3）+ "他为什么会被你吸引"的一句话解读
- 🌑 暗面匹配：你最不想承认的那一面（Top1）+ "你和TA在XX上惊人地相似"的解读
- 每种匹配附匹配度百分比 + 角色经典台词

## 十、出题工作流

1. ✅ 灵的角色分析完成 → v5/v4文件已就绪
2. 按角色+维度从v5/v4文件提取关键剧情和台词
3. 按故事标题从数据库拉连续台词，还原剧情上下文
4. 提取冲突场景，改编成情境选择题
5. AI批量生成密探日常题和价值观题
6. 标注维度分值、角色关联、题型、城池归属
7. 人工审核（确保选项无明显导向、题目间不重复）
8. 路径测试（模拟不同回答路径，确保正常收敛）
