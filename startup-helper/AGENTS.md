## ⚠️ 流水线执行纪律（强制，不可跳过）

当协调者（main agent）派发流水线任务给你时，**必须严格执行，不得偷懒、跳步、简化**。

### 铁律（违反 = 任务失败）

1. **逐项执行，禁止跳过** — 收到的任务清单、冒烟验证清单，必须逐项执行并记录结果。不允许"看起来没问题"这种空话。
2. **无文件 = 没执行** — 所有产出必须写入指定路径的文件。内存里的"我做了"不算数。
3. **验证必须带证据** — 冒烟验证的每一项，必须记录：验证步骤、预期结果、实际结果、通过/失败。
4. **禁止空签收** — 评审/签收报告必须包含至少3个具体检查点和实际检查内容。"看起来OK，签收" = 无效签收。
5. **遇到问题必须立即上报** — 编译失败、测试挂了、环境问题 → 立即报告协调者，禁止自行决定跳过。
6. **不自行决定跳过任何阶段** — 即使觉得某阶段"没必要"，也要执行并报告结果。跳过阶段的权力只在协调者和用户手上。

### 执行报告格式（强制）

每次完成任务后，必须输出：

```
✅/❌ [任务名]
- 执行步骤：[具体做了什么]
- 产出文件：[路径 + 行数]
- 验证结果：[逐项列出]
- 遇到的问题：[有就写，没有写"无"]
```

### 防偷懒检查

- 产出物行数不足要求最低线（PRD<200行、架构<100行、任务清单<50行）→ 自动打回
- 验证报告没有逐项结果 → 自动打回
- 没有写入文件 → 自动打回
- 连续2次被指执行不力 → 升级到协调者汇报用户
