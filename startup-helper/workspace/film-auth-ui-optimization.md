## 二、五项核心优化

### 优化 1：灰调去冷 → 暖灰体系

#### 问题
原色板 `#0D0D12`、`#1C1C2A`、`#2E2E42` 含蓝紫分量。冷色调长时间观看产生"工业冷感"——像监控室而不是创作空间。影视工具偏暖更舒适。

#### 优化方案

| 层级 | 旧值 | 新值 | 变化说明 |
|------|------|------|---------|
| Surface-0 | `#0D0D12` | `#0D0D0F` | 去除蓝分量 |
| Surface-1 | `#14141F` | `#151518` | 微暖 |
| Surface-2 | `#1C1C2A` | `#1E1E22` | 去紫 |
| Surface-3 | `#242436` | `#27272C` | 中性偏暖 |
| Surface-4 | `#2E2E42` | `#313138` | 保持对比 |
| Text-Primary | `#F0F0F5` | `#F2F0ED` | 微暖白 |
| Text-Secondary | `#A0A0B8` | `#A8A4A0` | 暖灰 |
| Text-Tertiary | `#6B6B82` | `#6E6A66` | 暖灰 |

#### 新增：暖调叠加层

| 名称 | 色值 | 用途 |
|------|------|------|
| Surface-Warm | `#E8753A@3%` | 选中态底色、播放器区域底色 |
| Brand-Hover | `#E8753A@8%` | hover/聚焦态微暖底色 |

**关键区别：不是给灰色加橙，而是在灰色上叠一层极低透明度的橙色。** 这让界面"有一点点暖"，但不是明显变色。

#### 对比示意

```
旧：#1C1C2A (R:28 G:28 B:42) → 蓝紫倾向明显
新：#1E1E22 (R:30 G:30 B:34) → 中性，R/G略高于B → 暖
+ Surface-Warm 叠加后 → 微微偏暖，像纸张的象牙白在暗色中的投射
```

#### 为什么不选更暖的灰？
过度暖灰（如 `#1E1E1C`）会显得"脏"。暖度的目标是"感觉不到，但比冷灰舒服"。

### 优化 2：审批页视频优先布局

#### 问题
原设计播放器和审批信息 50/50 分割。审批的核心动作是"看片"，播放器是主角，审批面板是配角。50/50 让视频不够大、审批面板太空。

#### 优化方案：65/35 分割

```
旧布局（50/50）：
┌──────────────────────┬──────────────────┐
│                      │                  │
│    播放器 50%        │  审批面板 50%     │
│                      │                  │
└──────────────────────┴──────────────────┘

新布局（65/35）：
┌────────────────────────────────┬──────────┐
│                                │          │
│         播放器 65%             │ 审批 35% │
│                                │          │
│   ▶ ━━━━━●━━━━ 01:23/02:35    │          │
└────────────────────────────────┴──────────┘
```

#### 右侧面板精简

只保留核心信息，去掉装饰性内容：

```
┌──────────────────┐
│ 审批信息（折叠）   │  ← 默认收起，点击展开
│ · 提交: 张三      │
│ · 版本: v3       │
│ · 时长: 02:35    │
├──────────────────┤
│                  │
│ [✅ 通过]        │  ← 三个按钮，大且醒目
│ [⚠️ 要求修改]    │
│ [❌ 拒绝]        │
│                  │
├──────────────────┤
│ 批注输入          │
│ ┌──────────────┐ │
│ │              │ │
│ └──────────────┘ │
│ [提交]           │
└──────────────────┘
```

#### 进度条强化

播放器底部的进度条是高频交互区，需要：
- 默认高度 4px，hover 时扩展到 8px
- 已播部分用 Brand-Primary 色
- 未播部分用 Surface-4
- 批注标记点在进度条上用小黄点显示

### 优化 3：项目时间轴

#### 问题
影视制作是时间线驱动的工作流——从前期到后期到交付，每一步都在时间轴上。但原页面结构是静态的"管理后台"式布局，缺乏"时间维度"的视觉表达。

#### 优化方案：项目阶段进度线

在项目详情页和审批页底部加一条极简的进度时间轴：

```
● 前制 ─── ● 拍摄 ─── ● 后期(当前) ─── ○ 审批 ─── ○ 交付
 │          │          │               │          │
 ✓          ✓          ▶              待开始      待开始
```

#### 设计规范

| 元素 | 规格 |
|------|------|
| 节点大小 | 8px 圆形 |
| 已完成节点 | Brand-Primary 实色填充 |
| 当前节点 | Brand-Primary 填充 + 脉冲动画（2s cycle） |
| 未来节点 | Surface-4 边框，无填充 |
| 连接线 | 2px，已完成段 Brand-Primary，未完成段 Surface-4 |
| 文字 | Caption 字号（12px），Text-Secondary |
| 当前阶段文字 | Text-Primary + "当前"标签 |

#### 放置位置

- **项目详情页**：放在页面标题下方，面包屑之后，主内容之前
- **审批页**：放在播放器和版本历史之间
- **工作台**：每个项目卡片的底部迷你时间轴（省略文字，只保留节点+线）

#### 为什么是线性而不是环形/网格？
影视制作是严格的线性流程。线性时间轴最直观地表达了"我们在哪、还剩什么"。

### 优化 4：信息密度控制

#### 问题
看板页 4 列 + 每列多卡片 + 标签 + 头像 + 日期，信息量爆炸。影视工作者经常连续看素材几小时，信息过载会加速疲劳。

#### 优化方案

| 页面 | 默认显示数量 | 超出处理 |
|------|------------|---------|
| 看板 | 每列 4 张卡片 | "展开更多" 按钮 |
| 项目列表 | 12 张卡片 | 分页（不是无限滚动） |
| 审批列表 | 20 条 | 分页 |
| 任务列表 | 30 条 | 分页 + 虚拟滚动 |

#### 信息分层原则

```
第一眼（1秒内）：状态标签 + 标题 + 缩略图
第二层（hover）：负责人 + 截止日期 + 进度
第三层（点击）：完整详情页
```

不要在一开始就把所有信息堆在卡片上。用户需要的是快速扫描，不是逐字阅读。

### 优化 5：影视身份感

#### 问题
整体像通用 SaaS 的暗色版。需要一些"只有影视工具才有的"视觉元素，建立行业辨识度。

#### 优化方案

**5a. 视频播放器发光边**

播放器区域下边缘 1px `#E8753A@15%` 发光线：

```css
.video-player::after {
  content: '';
  position: absolute;
  bottom: 0;
  left: 0;
  right: 0;
  height: 1px;
  background: linear-gradient(90deg, transparent, rgba(232,117,58,0.15), transparent);
}
```

这不是渐变（整个页面的原则），这是播放器专属的"聚焦提示"。

**5b. 版本标记用电影胶片齿孔图标**

版本历史不用普通的 `v1 | v2 | v3`，用胶片齿孔样式的图标：

```
[v1] ○──○──[v2] ○──○──[v3●]
     ▒▒      ▒▒      ▒▒
```

**5c. 空状态用影视相关 SVG 插图**

| 页面 | 插图内容 |
|------|---------|
| 项目列表空 | 场记板 + 空镜头 |
| 审批列表空 | 躺椅 + 爆米花（"暂时没有要看的片子"） |
| 任务看板空 | 剪辑台 + 空白胶卷 |
| 素材库空 | 灯箱 + 空相框 |

**5d. 加载状态用影视元素**

- 骨架屏 → 电影胶片旋转动画
- 视频上传进度 → 胶卷轴转动
- 项目创建 → 场记板打板动画
