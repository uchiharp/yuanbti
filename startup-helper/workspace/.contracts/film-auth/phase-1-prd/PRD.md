# 统一身份认证与权限管理系统 PRD

## 1. 产品概述

### 1.1 产品定位

为影视公司提供统一的身份认证（SSO）和权限管理平台。所有业务系统通过一套账号体系实现单点登录，权限统一管控。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 单点登录 | 一次登录，访问所有业务系统 |
| 统一账号 | 用户在任意系统注册/修改，全局生效 |
| 细粒度权限 | 功能级 + 项目级 |
| 可扩展 | 新业务系统低成本接入 |

### 1.3 用户规模

- 初期：500-1000 用户
- 目标：支撑 5000+ 用户
- 并发：100-500 QPS

## 2. 用户角色

### 2.1 系统管理员

- 管理所有用户账号
- 配置角色和权限
- 查看审计日志

### 2.2 项目负责人

- 管理项目成员
- 分配项目内角色
- 查看项目相关数据

### 2.3 普通用户

- 登录各业务系统
- 在权限范围内操作
- 管理个人信息

### 2.4 外部协作方

- 受限访问特定项目
- 有时间限制的临时账号
- 只能查看不能修改

## 3. 功能需求

### 3.1 单点登录（SSO）

#### 3.1.1 登录方式

| 方式 | 版本 | 说明 |
|------|------|------|
| 账号密码 | V1.0 | 支持用户名登录 |

#### 3.1.2 SSO 协议

- **OAuth 2.0 授权码模式**：业务系统跳转认证中心登录，拿到授权码换 token
- **JWT Token**：无状态认证，各系统本地验签

#### 3.1.3 Token 机制

```
access_token:  有效期 2 小时，用于 API 调用
refresh_token: 有效期 7 天，用于刷新 access_token
```

- access_token 过期后，业务系统用 refresh_token 静默刷新
- 用户修改密码后，所有 token 失效
- 支持强制下线（管理员踢人）

#### 3.1.4 Token 黑名单（立即踢出）

任何影响用户权限/状态的操作，都应立即踢出相关用户。

**触发自动踢出的操作：**

| 触发操作 | 踢出范围 | 说明 |
|----------|---------|------|
| 禁用用户 | 该用户 | 所有 token 立即失效 |
| 强制下线 | 该用户 | 所有 token 立即失效 |
| 删除用户 | 该用户 | 所有 token 立即失效 |
| 修改密码 | 该用户 | 旧 token 全部失效 |
| 分配角色 | 该用户 | 踢出后重新加载权限 |
| 移除角色 | 该用户 | 踢出后重新加载权限 |
| 修改角色权限 | 该角色所有用户 | 权限变了，全员踢出 |
| 禁用角色 | 该角色所有用户 | 角色不可用，全员踢出 |
| 移除项目成员 | 该用户 | 项目权限被收回 |

**批量踢出优化：** 修改角色权限时，需要批量踢出相关用户，避免逐个操作。

#### 3.1.5 业务系统接入流程

```
1. 用户访问业务系统（如项目管理系统）
2. 业务系统检测到未登录，跳转到认证中心登录页
3. 用户在认证中心登录（或已登录则跳过）
4. 认证中心回调业务系统，携带授权码 code
5. 业务系统用 code 换取 access_token
6. 业务系统用 access_token 获取用户信息
7. 用户进入业务系统
```

#### 3.1.6 第三方应用注册

| 字段 | 说明 |
|------|------|
| client_id | 应用唯一标识 |
| client_secret | 应用密钥 |
| redirect_uri | 回调地址（白名单） |
| scope | 请求的权限范围 |
| 名称 | 应用显示名称 |
| 图标 | 应用图标 URL |

### 3.2 用户管理

#### 3.2.1 用户字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | ✅ | 登录名，唯一 |
| password | string | ✅ | 加密存储 |
| nickname | string | ✅ | 显示名称 |
| email | string | ❌ | 邮箱 |
| phone | string | ❌ | 手机号 |
| avatar | string | ❌ | 头像 URL |
| status | enum | ✅ | 正常/禁用/待激活 |
| created_at | datetime | auto | 创建时间 |
| last_login_at | datetime | auto | 最后登录时间 |

#### 3.2.2 用户操作

- 创建用户（管理员）：单个创建 / 批量创建
- 批量创建：文本框输入用户名，英文逗号分隔，系统自动去除前后空白/制表符
- 用户名规则：只允许中文、英文、数字
- 自动生成密码：创建用户时系统自动生成随机密码，首次登录强制修改
- 修改用户信息
- 启用/禁用用户（禁用后立即踢出）
- 重置密码（生成新密码，首次登录强制修改）
- 删除用户（软删除）

**不支持：** 用户自助注册（V1.0 不开放）

#### 3.2.3 密码策略

- 最小长度：8 位
- 必须包含：大小写字母 + 数字
- 密码过期：90 天强制修改（可配置）
- 登录失败锁定：连续 5 次失败锁定 30 分钟
- 禁止使用最近 3 次的密码
- 自动生成密码：12 位随机密码（含大小写字母+数字+特殊字符）
- 首次登录：系统生成的密码必须在首次登录时修改

### 3.3 角色管理

#### 3.3.1 预设角色

| 角色 | 说明 | 权限范围 |
|------|------|---------|
| super_admin | 超级管理员 | 所有系统全部权限，不可删除 |

其他角色由管理员自行创建并分配权限。

**权限分配规则：**
- 超级管理员：拥有所有权限，无需手动分配
- 其他角色：管理员按需创建，分配具体权限
- 一个用户可拥有多个角色，权限取并集

#### 3.3.2 自定义角色

- 管理员可创建自定义角色
- 角色字段：编码、名称、描述、权限列表
- 支持角色继承（可选）

#### 3.3.3 权限管理

**权限格式：** `系统:模块:功能点`（三级结构）

```
film-project:project:create          影视项目系统 → 项目模块 → 创建
film-project:project:view            影视项目系统 → 项目模块 → 查看
film-project:project:edit            影视项目系统 → 项目模块 → 编辑
film-project:task:create             影视项目系统 → 任务模块 → 创建
film-project:task:assign             影视项目系统 → 任务模块 → 分配
film-asset:asset:upload              资产系统 → 资产模块 → 上传
film-asset:asset:download            资产系统 → 资产模块 → 下载
film-video:video:generate-720p       视频系统 → 生成模块 → 生成720p
film-video:video:generate-1080p      视频系统 → 生成模块 → 生成1080p
film-video:video:generate-4k         视频系统 → 生成模块 → 生成4k
film-video:video:preview             视频系统 → 预览模块 → 预览
film-finance:budget:view             财务系统 → 预算模块 → 查看
film-finance:budget:approve          财务系统 → 预算模块 → 审批
auth:user:manage                     认证中心 → 用户模块 → 管理
auth:role:manage                     认证中心 → 角色模块 → 管理
auth:audit:view                      认证中心 → 审计模块 → 查看
```

**三级命名灵活扩展：** 功能点（第三级）可按需细粒度拆分，例如：
- `generate-720p` / `generate-1080p` / `generate-4k` — 按画质拆分
- `download-hd` / `download-sd` — 按清晰度拆分
- 不拆分时用通用名如 `create`、`view`、`download`

**权限分类（模块化）：**

| 分类 | 说明 | 示例 |
|------|------|------|
| 系统权限 | 认证中心自身功能 | auth:user:manage, auth:role:manage |
| 业务权限 | 各业务系统功能 | film-project:project:create |
| 数据权限 | 数据范围控制 | own / project / all |

**权限分组显示：** 前端按 `系统` 分组展示，方便按业务系统勾选。

**权限分配方式：**

| 方式 | 说明 |
|------|------|
| 交互式勾选 | 按系统+模块分组展示，勾选添加 |
| JSON 导入 | 输入 JSON 数组批量导入 |
| 角色复制 | 从已有角色复制权限 |
| 权限模板 | 预定义常用权限组合，一键应用 |

**权限模板（创建角色时可选用）：**

| 模板名 | 权限集 |
|--------|--------|
| 项目管理员 | film-project:project:*, film-asset:asset:* |
| 内容编辑 | film-video:video:generate-*, film-asset:asset:upload, film-asset:asset:download |
| 只读用户 | film-project:project:view, film-asset:asset:download |
| 财务人员 | film-finance:budget:view, film-finance:budget:approve |

### 3.4 项目级权限

#### 3.4.1 项目成员

| 项目角色 | 权限 |
|----------|------|
| owner | 项目全部权限 + 成员管理 |
| editor | 项目内编辑权限 |
| viewer | 项目内只读权限 |

#### 3.4.2 数据范围（Data Scope）

用户在不同项目中可以有不同的数据范围：

| 范围 | 说明 | 示例 |
|------|------|------|
| own | 只看自己创建/负责的数据 | 剪辑师只看自己负责的任务 |
| project | 看所在项目的所有数据 | 项目负责人看项目内所有任务 |
| all | 看全部数据（跨项目） | 管理员看所有项目 |

**权限计算：**

```
最终权限 = 角色权限 ∩ 项目权限 ∩ 数据范围

例：用户是 editor 角色（有 video:generate 权限），
   在项目 A 中是 viewer（无编辑权限），
   则在项目 A 中无法生成视频。

例：用户有 task:view 权限，数据范围是 own，
   则只能看到自己负责的任务，看不到项目内其他人的任务。
```

**数据范围配置：**

- 角色可预设默认数据范围（如 editor 默认 `project`，viewer 默认 `own`）
- 管理员可在分配角色时覆盖默认范围
- 支持批量设置：选中多个用户，统一设置数据范围

#### 3.4.3 数据隔离

- 用户只能看到自己参与的项目（除非数据范围是 `all`）
- 管理员可查看所有项目

### 3.7 API 接口

#### 3.7.1 接口分类

接口按使用场景分为两类：

| 类型 | 调用方 | 协议 | 说明 |
|------|--------|------|------|
| 浏览器接口 | 用户浏览器 | HTTP | SSO 登录流程，涉及页面跳转 |
| 服务间接口 | 业务系统后端 | HTTP / RPC | 权限校验、获取用户信息等 |

服务间接口同时提供 HTTP 和 RPC 两种协议，业务系统按技术栈选择。

#### 3.7.2 浏览器接口（SSO 登录流程）

| 接口类型 | 说明 |
|----------|------|
| 登录页 | 提供登录页面，接收账号密码 |
| 登录提交 | 验证凭据，返回 token |
| OAuth 授权 | 业务系统跳转到认证中心，返回授权码 code |
| OAuth 回调 | 业务系统用 code 换取 access_token |
| 刷新 Token | 用 refresh_token 换取新的 access_token |
| 注销 | 作废当前 token |

#### 3.7.3 服务间接口（业务系统调用）

| 接口类型 | 说明 |
|----------|------|
| Token 验证 | 验证 token 是否有效，返回用户基本信息 |
| 获取用户信息 | 返回用户完整信息 + 角色列表 + 权限列表 |
| 权限校验 | 检查用户是否有某权限，返回允许/拒绝 |

**权限校验说明：**
- 业务系统传入「要检查的权限」（如 `video:generate-1080p`）
- 系统返回「是否允许」
- 用于各业务系统在执行操作前做权限判断

#### 3.7.2 用户管理接口

| 接口 | 说明 | 权限 |
|------|------|------|
| 用户列表 | 分页查询，支持按状态/关键词筛选 | user:manage |

| 创建用户 | 手动创建，系统自动生成密码 | user:manage |

| 启用/禁用 | 切换用户状态，禁用自动踢出 | user:manage |
| 批量创建 | 文本框输入用户名（逗号分隔），自动去空白，用户名只允许中文/英文/数字 | user:manage |

### 假设清单

- [V1.0仅账号密码登录，用户接受] → 若需其他登录方式则需V2.0提前
- [用户规模500-5000+，QPS<500] → 若超出需引入分布式session/缓存集群
- [业务系统均为自研，可改造接入SSO] → 若有第三方系统需V1.1 OAuth2标准协议
- [所有服务部署在内网或通过HTTPS访问] → 若HTTP部署则JWT有泄露风险
- [数据库使用PostgreSQL（已有阿里云实例）] → 若换数据库需调整migration脚本
- [前端使用React+TypeScript] → 若换框架需重写前端
- [Token黑名单基于Redis实现] → Redis不可用则降级为数据库轮询，性能下降
- [权限校验由认证中心统一提供API] → 各业务系统需实现本地缓存，否则每次请求都调认证中心

### V2.0

- 审批流
- LDAP/AD 对接
- 多租户支持
- OAuth2/OIDC 标准协议 — 让第三方应用（非你们开发的系统）安全接入认证中心，用户无需把密码给第三方，通过授权页面控制权限范围
