# 阶段4：UI 设计

## 任务
基于 UX 设计，产出视觉设计方案。

## 必读文件
1. UX-DESIGN.md
2. PRD.md

## 产出物
| 文件 | 说明 |
|------|------|
| UI-DESIGN.md | 视觉设计方案 |
| ui-review.md | PM + 开发审查意见 |
| re-review-ui.md | PM + 开发复审确认 |

## 执行流程
1. 读 UX-DESIGN.md + PRD.md
2. 产出 UI-DESIGN.md
3. PM 审查：页面覆盖度、交互完整性、需求符合度、逻辑 bug
4. 开发审查：技术可行性、组件复用性、响应式适配、逻辑 bug
5. UI 根据意见修改
6. **PM + 开发复审**：确认问题已修复
7. 复审通过 → 推进阶段 4.5；仍有问题 → UI 再改（最多 2 轮）

## 约束
- 总轮次 ≤2
