# 阶段1.5：PRD 交叉评审

## 任务
各角色并发评审 PRD，从不同视角提出意见。

## 必读文件
1. PRD.md

## 加载 Skill
无（各角色用自己的专业视角评审）

## 产出物
| 文件 | 最低行数 | 说明 |
|------|---------|------|
| cross-review-pm.md | 50 | 每个参与角色独立段落 ≥20行，每段 ≥3检查点 |

## 执行流程
1. 协调者并发调度各角色 agent
2. 各角色独立评审：
   - 架构师：技术可行性？性能隐患？
   - QA：验收标准具体？能写测试用例？
   - 开发：工作量合理？边界遗漏？
3. PM 汇总意见 → 修订 PRD → PM·评审官复核（1轮）

## 检查项（脚本强制）
- [ ] cross-review-pm.md 存在
- [ ] 每个参与角色有独立 ## 段落
- [ ] 每段 ≥3 检查点
- [ ] 修订后的 PRD.md 行数仍 ≥200
- [ ] communications/ 有评审记录

## 约束
- 时限：24h（脚本记录起始时间）
- 并发评审，不做迭代，只提意见
- 仅 🟡中型 和 🔴大型 项目执行此阶段
