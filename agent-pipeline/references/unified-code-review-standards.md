## ⚠️ 关键声明：本清单不是全部

这份标准是你的**最低要求**，不是**全部要求**。

**你必须自由发挥，关注标准之外的任何问题**，包括但不限于：
- 代码里"感觉不对"但说不上来为什么的地方
- 看着能跑但实际上逻辑有隐患的写法
- 用户实际使用时可能遇到的奇怪情况
- 你作为"对手"觉得原Agent可能偷懒的地方
- 行业最佳实践里提到但这份清单没写的东西

**如果只有这份清单里写了的问题才被你发现 = 你不合格。**
真正优秀的评审官能发现清单之外的问题。

# 统一代码审查标准 v2.0

## 评分体系

| 等级 | 条件 |
|------|------|
| 10分 | 几乎不可能，代码本身就是教材级 |
| 9分 | 清单外有独立发现，且全部已修复 |
| 8分 | 无🔴🟡问题，🟢≤3个 |
| 7分 | 默认基准线，有少量问题但可接受 |
| 6分 | 🟡≤3个，全部已修复 |
| 5分 | 存在未修复🟡，或🟢>5个 |
| 4分 | 🔴1个（非一票否决类），已修复 |
| 3分 | 🔴1个未修复 |
| 2分 | 多个🔴 |
| 1分 | 一票否决触发 |

### 一票否决（直接判0分）

- SQL注入漏洞
- 数据丢失风险（删除无确认、事务未提交直接返回、truncate误用）
- 事务边界错误（只读操作开启写事务、事务内调用外部API）
- 敏感信息泄露（密码明文、密钥硬编码、日志打印token）
- 前端白屏（未处理异步错误、未捕获Promise rejection）
- `@author AI` / `@author Claude` / `@author GPT` 等AI署名

### 问题分级

| 级别 | 标记 | 含义 | 要求 |
|------|------|------|------|
| 阻断 | 🔴 | 必须修复才能合并 | 打回重做 |
| 重要 | 🟡 | 强烈建议修复 | 需说明不修的理由 |
| 建议 | 🟢 | 可以改进 | 记录跟踪 |

# 附录A：一眼AI套路速查表

## EYE级（代码层面，每条-3分）

| 编号 | 模式 | 示例 |
|------|------|------|
| EYE-01 | 万能service命名 | `UserService`/`OrderService`只做CRUD无业务逻辑 |
| EYE-02 | 注释复读机 | `// 获取用户名` `public String getUsername()` |
| EYE-03 | 过度封装 | 简单if-else拆成3个类+1个接口+1个工厂 |
| EYE-04 | 无意义DTO | `UserVO`和`UserDTO`字段完全一样 |
| EYE-05 | 链式Builder滥用 | 5个字段的PO也用Builder模式 |

## SUS级（可疑特征，每条-2分）

| 编号 | 模式 |
|------|------|
| SUS-01 | 文件头注释格式与项目其他文件不一致 |
| SUS-02 | import顺序与IDE自动排列不同（AI有自己的import习惯） |
| SUS-03 | 换行风格不一致（有时100字符换行，有时不换） |
| SUS-04 | 异常消息措辞过于正式（"无法完成请求的操作" vs "订单不存在"） |
| SUS-05 | 变量命名风格漂移（局部突然用了`result`/`data`这类AI常用名） |
| SUS-06 | 完全没有git blame历史（一次性提交大量代码） |
| SUS-07 | 代码结构过于对称（每个方法长度差不多、注释密度均匀） |

## DOC级（文档层面，每条-3分）

| 编号 | 模式 |
|------|------|
| DOC-01 | Javadoc中有"该方法用于..."这种AI常用句式 |
| DOC-02 | README.md风格与项目其他文档明显不同 |
| DOC-03 | 参数说明直接复制参数名（`@param userId 用户ID`） |
| DOC-04 | 变更日志格式化得过于整齐 |
| DOC-05 | 代码示例过于完美（无真实业务场景的痕迹） |
| DOC-06 | 技术选型理由写得像论文（有references但没有真实踩坑经历） |
| DOC-07 | 性能描述用模糊词汇（"显著提升"、"大幅优化"而无具体数据） |

## 三眼识别（一票否决）

同时命中以下3条中的任意组合 = AI生成基本确定 = 一票否决：

1. **代码指纹**：EYE级命中≥3条
2. **文档指纹**：DOC级命中≥2条
3. **提交指纹**：单次commit超500行且commit message是AI风格（`feat: implement complete user management module with full CRUD operations and comprehensive error handling`）
